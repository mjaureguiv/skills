# User Interview Skills

A comprehensive skill suite for SAP Signavio PMs to conduct effective user interviews at every stage of the product development cycle. Based on the MxG PM Learning Series Discovery Enablement session.

## Overview

This skill suite helps PMs ask the right questions at the right time, following the double-diamond model of product development:

```
Discovery → Problem Validation → Solution Validation → Solution Refinement → Post-Launch
(Diverge)      (Converge)           (Diverge)             (Converge)          (Learn)
```

## Quick Start

1. **Identify your phase** - Where are you in the product development cycle?
2. **Use the right skill** - Each phase requires different questioning techniques
3. **Prepare your context** - Know your research goals before the interview
4. **Practice** - Use the sample questions as templates, not scripts

## Key Principles (Apply to All Phases)

### Do's
- Be neutral and curious about all feedback (especially negative)
- Encourage users to share their own opinions
- Reflect questions back to understand reasoning
- Probe deeper into comments and behaviors
- Start broad, then narrow down
- Follow the 80:20 rule (listen 80%, speak 20%)
- Ask one question at a time
- Ask about specific moments, not habits
- Aim for 8-10 quality interviews minimum

### Don'ts
- Don't become defensive or sell your product
- Don't bias users with leading language
- Don't explain how the product works before understanding why they asked
- Don't accept surface-level responses without probing
- Don't begin with closed or leading questions
- Don't dominate the conversation

## Question Framing Tips

| Instead of... | Try... |
|---------------|--------|
| "How satisfied are you with SAP products?" | "How would you rate your experience with SAP products?" |
| "How do you usually react to unhelpful messages?" | "How did you react the last time you saw error message XYZ?" |
| "How happy were you using our product?" | "How did you feel using our product?" |
| "What activities did you perform AND would those be typical?" | Ask each question separately |

## Sample Interview Flow

```
1. Warm-up (2-3 min)
   - Thank them for their time
   - Brief intro to the session purpose
   - "No wrong answers"

2. Context Setting (5 min)
   - Role and responsibilities
   - Typical day overview
   - Relationship with the tool/process

3. Core Questions (30-40 min)
   - Use phase-specific questions
   - Follow interesting threads
   - Ask for specific examples

4. Wrap-up (5 min)
   - "Anything else we should know?"
   - "If you could change one thing..."
   - Thank you and next steps
```
