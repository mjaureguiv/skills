# Post-Launch Interviews

Understand adoption patterns, long-term value, and expansion opportunities.

## When to Use

- Feature has been live for 1+ months
- Usage data shows inconsistent engagement
- Some users are power users, others barely touch it
- You need to understand why adoption varies
- Looking for improvement or expansion opportunities

## What This Does

Helps you:
- Generate interview guides for active vs. inactive users
- Understand what differentiates power users from churned users
- Identify barriers to adoption and opportunities for expansion
- Validate long-term value and stickiness

## How to Use

**Generate a guide:**
```
/interview-postlaunch

Tool: [Tool/feature name]
Live for: [Duration]
Pattern: [What you're seeing in usage data]

Help me create interview guides.
```

**Analyze adoption patterns:**
```
/interview-postlaunch

Help me understand why some users adopt [feature] while others don't.
Here's what I found in interviews: [findings]
```

## Key Principles

### DO
- Interview BOTH active and inactive users
- Let usage data inform (but not bias) your questions
- Ask "If we removed this tomorrow..."
- Look for patterns that differentiate segments

### DON'T
- Only talk to happy customers
- Assume you know why inactive users churned
- Ask leading questions about specific features
- Skip the stickiness test

## Sample Questions

**For Active Users:**
| Type | Question |
|------|----------|
| **Value** | "What's the most valuable part for you?" |
| **Impact** | "How has this changed your workflow?" |
| **Stickiness** | "If we removed this tomorrow, what would you do?" |
| **Expansion** | "What would make you use it more?" |

**For Inactive Users:**
| Type | Question |
|------|----------|
| **Experience** | "Tell me about when you tried this tool." |
| **Drop-off** | "What made you stop using it?" |
| **Expectations** | "What did you expect it would do?" |
| **Recovery** | "What would need to change for you to try again?" |

## User Segmentation

| Segment | Focus Areas |
|---------|-------------|
| **Power Users** | Value, expansion, testimonials |
| **Occasional Users** | Barriers to more use |
| **Churned Users** | Why they left |
| **Never Engaged** | Initial adoption barriers |

## Expected Output

After 8-10 interviews (mixed segments):
- Clear adoption pattern explanation
- Value validation from power users
- Barrier identification from inactive users
- Expansion opportunities
- Input for next discovery cycle

Findings often loop back to [Problem Discovery](../problem-discovery/) for the next iteration.
