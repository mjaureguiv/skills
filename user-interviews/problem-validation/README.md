# Problem Validation Interviews

Validate which problems are painful, frequent, and worth solving.

## When to Use

- You've completed Problem Discovery and have 7-10 potential problems
- Leadership needs to know which problem to invest in
- You need evidence to prioritize the backlog
- You're deciding between multiple opportunity areas

## What This Does

Helps you:
- Generate interview guides for problem validation
- Review questions for evidence vs. opinion gathering
- Use the "$10 allocation" prioritization technique
- Synthesize findings into prioritization decisions

## How to Use

**Generate a guide:**
```
/interview-problem

I have these problems from discovery:
1. [Problem A]
2. [Problem B]
3. [Problem C]

Help me create a validation guide.
```

**The $10 Exercise:**
```
/interview-problem

Help me set up a $10 allocation exercise for [problems].
```

## Key Principles

### DO
- Probe for evidence ("When was the last time...")
- Quantify impact (time, cost, effort)
- Compare across problems
- Use trade-off exercises

### DON'T
- Accept opinions without examples
- Mention solutions
- Ask leading questions
- Let them prioritize for you (gather data, you decide)

## Sample Questions

| Type | Question |
|------|----------|
| **Severity** | "What are the consequences when this happens?" |
| **Frequency** | "How often do you encounter this?" |
| **Evidence** | "Walk me through the last time..." |
| **Priority** | "If you had $10 to invest, how would you allocate?" |

## Expected Output

After 7-8 validation interviews:
- Clear top-priority problem with evidence
- Severity and frequency data for each problem
- Stakeholder-ready justification
- Problem statement ready for Solution Validation

Take the winning problem to [Solution Validation](../solution-validation/) next.
