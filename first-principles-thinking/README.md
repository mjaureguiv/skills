# First Principles Thinking

Break down complex problems to their fundamental truths and rebuild better solutions from the ground up.

## What This Does

Guides you through a structured first principles analysis: surface all assumptions, challenge each one, identify what's actually true, then build from there.

## When to Use

- You're stuck and conventional solutions aren't working
- You want to innovate, not just iterate
- You're questioning why something is done a certain way
- You need to challenge an industry assumption
- You want a non-obvious solution to a well-known problem

## How to Use

Just state your problem:

```
Why do we do discovery the way we do? I feel like we're just going through the motions.
```

```
Our sprint velocity keeps dropping. Let's break this down from first principles.
```

```
Is a 6-week discovery phase actually necessary for this feature?
```

## The Process

1. **Define** — State the problem clearly
2. **Uncover** — List all embedded assumptions
3. **Challenge** — Question each assumption: law of physics or just convention?
4. **Identify truths** — What remains undeniably true?
5. **Rebuild** — Design a solution with only fundamental truths as constraints

## Output

A structured analysis:

```markdown
## First Principles Analysis: [Problem]

### Assumptions Challenged
| Assumption | Type | Verdict |
|------------|------|---------|
| [assumption] | Convention/Constraint | Breakable/Fixed |

### Fundamental Truths
1. [Truth that cannot be violated]

### New Possibilities
1. [Solution that becomes possible]

### Recommended Next Step
[Smallest experiment to test]
```
