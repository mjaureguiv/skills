## Clarification of Distinct Boundaries

**#3 (Granular Admin Responsibilities) vs #6 (Custom Role Definition)**
- #3: Separating duties for existing admins (user mgmt admin, config admin, approval admin) - LEAST PRIVILEGE
- #6: Creating new role types and customizing built-in roles - FLEXIBILITY

**#4 (Self-Service Account) vs #3 (Granular Admin)**
- #4: Users and admins managing account STATUS (unlock, retention, lockout) - ACCOUNT MANAGEMENT
- #3: Admins having LIMITED duties (not all admin functions) - RESPONSIBILITY SEPARATION

**#5 (Data Access Control) vs #8 (Publishing Governance)**
- #5: Who can READ/VIEW data and cases (visibility) - DATA VISIBILITY
- #8: Who can PUBLISH/MODIFY content (modification rights) - CONTENT MODIFICATION

**#1 (Offboarding) vs #2 (Bulk Operations)**
- #1: Specific workflow for user departure (dependencies, handover, replacement) - DEPARTURE WORKFLOW
- #2: Efficiency tool for bulk add/remove (any scenario) - BULK ACTIONS
